## 1. Database Schema Overview

### Tables Involved

| Table | Purpose |
|---|---|
| `enrollments` | Created on every course purchase (single or inside bundle) |
| `bundle_enrollments` | Created when student buys a bundle |
| `faculty_transactions` | Created only at payout time — tracks sales + transfers |
| `platform_settings` | Stores admin commission %, other global config |

### Key Columns

**enrollments**
```
id                    → primary key
student_id            → who bought
course_id             → which course
faculty_id            → which faculty earns
amount_paid           → total paid by student (in paise)
gst_amount            → GST portion (in paise)
course_price          → original course price
is_bundle_enrollment  → true if came from bundle purchase
bundle_enrollment_id  → links to bundle_enrollments if is_bundle_enrollment=true
payment_id            → razorpay/payment gateway id
```

**bundle_enrollments**
```
id          → primary key
student_id  → who bought
bundle_id   → which bundle
faculty_id  → which faculty owns the bundle
amount_paid → total bundle amount paid (in paise)
payment_id  → razorpay/payment gateway id
```

**faculty_transactions**
```
id                    → primary key
faculty_id            → which faculty
transaction_id        → auto generated by trigger
type                  → COURSE_SALE | BUNDLE_SALE | PAYOUT | PLATFORM_FEE | REFUND
status                → PENDING | SUCCESS | FAILED
amount                → faculty share (COURSE_SALE/BUNDLE_SALE/PAYOUT) OR commission (PLATFORM_FEE)
gross_amount          → base amount before commission deduction
gst_amount            → GST amount for reference
commission_percent    → % rate used at time of payout (audit trail)
enrollment_id         → filled for COURSE_SALE rows
bundle_enrollment_id  → filled for BUNDLE_SALE rows
payout_id             → points to PAYOUT transaction_id (for COURSE_SALE, BUNDLE_SALE, PLATFORM_FEE rows)
transacted_at         → when this transaction happened
```

**platform_settings**
```
key    → 'default_commission_percent'
value  → '20'  (admin takes 20%)
```

## 2. Flow Overview

```
SINGLE COURSE:                        BUNDLE:
Student buys course                   Student buys bundle
        ↓                                     ↓
enrollments row created               bundle_enrollments row created
  is_bundle_enrollment = false          faculty_id, amount_paid stored
  amount_paid, gst_amount stored              ↓
        ↓                             [trigger fires automatically]
   [Nothing else]                     create_enrollments_on_bundle_purchase
                                              ↓
                                      N enrollments rows created
                                        is_bundle_enrollment = true
                                        bundle_enrollment_id = filled
                                        amount_paid = 0 (bundle price is on bundle_enrollments)
                                        ↓
                                       [Nothing else]

═════════════════════════════════════════════════
                  MONTHLY PAYOUT
═════════════════════════════════════════════════

Admin clicks "Process Payout" for a faculty
        ↓
Get commission % from platform_settings
        ↓
Find unpaid single enrollments
(enrollments with no COURSE_SALE in faculty_transactions)
        ↓
Find unpaid bundle enrollments
(bundle_enrollments with no BUNDLE_SALE in faculty_transactions)
        ↓
Calculate faculty share per enrollment
        ↓
Create COURSE_SALE row per single enrollment
Create BUNDLE_SALE row per bundle enrollment
Create PLATFORM_FEE row per enrollment (admin commission record)
Create ONE PAYOUT row (total transfer amount)
        ↓
Link all rows via payout_id
```

## 3. Track 1 — Single Course Purchase

### What happens
```
Student pays ₹1,180 for a course (₹1,000 + ₹180 GST)

enrollments row:
  amount_paid          = 118000  (paise)
  gst_amount           = 18000   (paise)
  is_bundle_enrollment = false
  bundle_enrollment_id = null
  faculty_id           = faculty-A
```

### At payout time
```
base_amount    = amount_paid - gst_amount  = 100000 paise (₹1,000)
commission     = base_amount * 20%         =  20000 paise (₹200)
faculty_share  = base_amount - commission  =  80000 paise (₹800)
```

### Rows created in faculty_transactions
```
Row 1 → COURSE_SALE:
  type               = COURSE_SALE
  amount             = 80000        ← faculty gets this
  gross_amount       = 100000       ← base after GST
  gst_amount         = 18000        ← GST excluded
  commission_percent = 20.00
  enrollment_id      = enroll-001
  payout_id          = TXN-PAY-001  ← links to PAYOUT row

Row 2 → PLATFORM_FEE:
  type               = PLATFORM_FEE
  amount             = 20000        ← admin keeps this
  gross_amount       = 100000
  gst_amount         = 18000
  commission_percent = 20.00
  enrollment_id      = enroll-001
  payout_id          = TXN-PAY-001
```

## 4. Track 2 — Bundle Purchase

### What happens
```
Student pays ₹2,500 for a bundle (3 courses inside)

bundle_enrollments row:
  amount_paid = 250000  (paise)
  faculty_id  = faculty-A
  bundle_id   = bundle-001

[trigger fires]

3 enrollments rows auto created:
  is_bundle_enrollment = true
  bundle_enrollment_id = bundle-enroll-001
  amount_paid          = 0   ← individual rows have 0, price is on bundle_enrollments
  faculty_id           = faculty-A
```

### At payout time
```
base_amount    = bundle amount_paid         = 250000 paise (₹2,500)
commission     = base_amount * 20%          =  50000 paise (₹500)
faculty_share  = base_amount - commission   = 200000 paise (₹2,000)

NOTE: Calculate commission on bundle_enrollments.amount_paid
      NOT on individual enrollment rows (those have amount_paid = 0)
```

### Rows created in faculty_transactions
```
Row 1 → BUNDLE_SALE:
  type                 = BUNDLE_SALE
  amount               = 200000           ← faculty gets this
  gross_amount         = 250000
  gst_amount           = 0                ← bundle may handle GST differently
  commission_percent   = 20.00
  bundle_enrollment_id = bundle-enroll-001
  enrollment_id        = null
  payout_id            = TXN-PAY-001

Row 2 → PLATFORM_FEE:
  type                 = PLATFORM_FEE
  amount               = 50000            ← admin keeps this
  gross_amount         = 250000
  commission_percent   = 20.00
  bundle_enrollment_id = bundle-enroll-001
  payout_id            = TXN-PAY-001
```

## 5. Monthly Payout Process

### Full Example

```
Faculty-A this month:
  Single course: 2 students → enroll-001 (₹1,000), enroll-002 (₹1,000)
  Bundle:        1 student  → bundle-enroll-001 (₹2,500)

Commission: 20%

Calculations:
  enroll-001:        base=₹1,000  commission=₹200   faculty=₹800
  enroll-002:        base=₹1,000  commission=₹200   faculty=₹800
  bundle-enroll-001: base=₹2,500  commission=₹500   faculty=₹2,000

  Total faculty payout = ₹800 + ₹800 + ₹2,000 = ₹3,600
```

### Rows created in faculty_transactions

```
┌──────────────┬────────┬─────────────┬───────────────┬──────────────────────┬─────────────┐
│ faculty_id   │ amount │ type        │ enrollment_id │ bundle_enrollment_id │ payout_id   │
├──────────────┼────────┼─────────────┼───────────────┼──────────────────────┼─────────────┤
│ faculty-A    │ 80000  │ COURSE_SALE │ enroll-001    │ null                 │ TXN-PAY-001 │
│ faculty-A    │ 20000  │ PLATFORM_FEE│ enroll-001    │ null                 │ TXN-PAY-001 │
│ faculty-A    │ 80000  │ COURSE_SALE │ enroll-002    │ null                 │ TXN-PAY-001 │
│ faculty-A    │ 20000  │ PLATFORM_FEE│ enroll-002    │ null                 │ TXN-PAY-001 │
│ faculty-A    │ 200000 │ BUNDLE_SALE │ null          │ bundle-enroll-001    │ TXN-PAY-001 │
│ faculty-A    │ 50000  │ PLATFORM_FEE│ null          │ bundle-enroll-001    │ TXN-PAY-001 │
│ faculty-A    │ 360000 │ PAYOUT      │ null          │ null                 │ TXN-PAY-001 │
└──────────────┴────────┴─────────────┴───────────────┴──────────────────────┴─────────────┘

amounts in paise
```

### payout_id linking
```
PAYOUT row transaction_id  = "TXN-PAY-001"
All other rows payout_id   = "TXN-PAY-001"

→ Query "show everything in March payout" = filter payout_id = "TXN-PAY-001"
```

## 6. Pending Payout Calculation

### Logic
```
Pending Payout =
  unpaid single enrollments (no COURSE_SALE transaction yet) after commission
  +
  unpaid bundle enrollments (no BUNDLE_SALE transaction yet) after commission

"Unpaid" = enrollment exists in enrollments/bundle_enrollments
           but has NO matching row in faculty_transactions
```

### Total Revenue vs Pending Payout

| Card | Source | Formula |
|---|---|---|
| Total Revenue | `enrollments` + `bundle_enrollments` | SUM of all `amount_paid` |
| Pending Payout | Unpaid enrollments only | SUM after GST and commission deduction |

## 7. Platform Settings

```sql
-- Default commission stored here
select value from platform_settings
where key = 'default_commission_percent';
-- returns '20'
```

### Commission resolution
```
Always fetch from platform_settings at payout time
Record the % used in commission_percent column on each transaction row
→ Future rate changes won't affect historical records
```

### Changing commission rate
```sql
update platform_settings
set value = '15'
where key = 'default_commission_percent';
-- Takes effect from next payout run
```

## 8. faculty_transactions Row Reference

| type | amount contains | enrollment_id | bundle_enrollment_id | payout_id |
|---|---|---|---|---|
| `COURSE_SALE` | faculty share | filled | null | filled |
| `BUNDLE_SALE` | faculty share | null | filled | filled |
| `PLATFORM_FEE` | admin commission | filled or null | filled or null | filled |
| `PAYOUT` | total transferred | null | null | own transaction_id |

5 Enrollments Example

enrollment-1: paid ₹1,180  (base ₹1,000 + GST ₹180)
enrollment-2: paid ₹2,360  (base ₹2,000 + GST ₹360)
enrollment-3: paid ₹590    (base ₹500  + GST ₹90)
enrollment-4: paid ₹1,180  (base ₹1,000 + GST ₹180)
enrollment-5: paid ₹3,540  (base ₹3,000 + GST ₹540)


gross_amount  gst_amount  commission%   amount
                     ─────────────────────────────────────────────────
COURSE_SALE  enr-1 →    1000          180         20.00        800
PLATFORM_FEE enr-1 →    1000          180         20.00        200

COURSE_SALE  enr-2 →    2000          360         20.00       1600
PLATFORM_FEE enr-2 →    2000          360         20.00        400

COURSE_SALE  enr-3 →     500           90         20.00        400
PLATFORM_FEE enr-3 →     500           90         20.00        100

COURSE_SALE  enr-4 →    1000          180         20.00        800
PLATFORM_FEE enr-4 →    1000          180         20.00        200

COURSE_SALE  enr-5 →    3000          540         20.00       2400
PLATFORM_FEE enr-5 →    3000          540         20.00        600



gross_amount = 1000 + 2000 + 500 + 1000 + 3000 = 7500  ← total base
gst_amount   = 180  + 360  + 90  + 180  + 540  = 1350  ← total GST
amount       = 800  + 1600 + 400 + 800  + 2400  = 6000  ← total transferred to faculty

                     gross_amount  gst_amount  commission%   amount
                     ─────────────────────────────────────────────────
PAYOUT row       →      7500         1350         20.00       6000



Verify it makes sense

Total student paid  = 7500 + 1350      = ₹8,850   (gross + GST)
Platform keeps      = 7500 × 20%       = ₹1,500   (commission)
Faculty gets        = 7500 - 1500      = ₹6,000   ✅ matches PAYOUT amount
GST goes to govt    = ₹1,350           (not anyone's income)

Cross check:
  SUM of all COURSE_SALE amounts  = 800+1600+400+800+2400 = 6000 ✅
  SUM of all PLATFORM_FEE amounts = 200+400+100+200+600   = 1500 ✅
  6000 + 1500                     = 7500 = gross_amount   ✅
